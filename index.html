Vavr.io : objectifs

  • Simplifier l'écriture de programmes fonctionnels en Java
  • Fournit des Collections immutables (Monades)
  • Fournit de nombreuses fonctions utilitaires
  • Fournit des outils et structures de contrôle

Vavr.io : fonctions communes

`.map` permet de transformer le contenu d'une monade.

map :: M a -> (a -> b) -> M b

						
							List<Integer> addOneToListElements(List<Integer> list) {
								return list.map(nb -> nb + 1);
							}

Vavr.io : fonctions communes

`.flatten` permet d'aplatir le contenu d'une monade.

flatten :: M (M a) -> M b

Vavr.io : fonctions communes

`.flatMap` permet de transformer le contenu d'une monade.

Equivalent à `.map().flatten()`

flatMap :: M a -> (a -> M b) -> M b

						
							List<Integer> reiterateListElements(List<Integer> list) {
								return list.flatMap(nb -> List.of(nb, nb));
							}

Vavr.io : fonctions communes

`.filter` permet de retirer des éléments d'une monade.

						
							List<Integer> getEvenNumbersFrom(List<Integer> list) {
								return list.filter(nb -> nb % 0 == 0);
							}

Vavr.io : quelques monades

Option<T>

  • Deux valeurs possibles : Some(T) et None
  • Permet de signifier la possibilité d'absence de valeur (remplace null).

Vavr.io : quelques monades

Either<E, T>

  • Deux valeurs possibles : Left(E) et Right(T)
  • Exprime le succès ou l'échec d'une fonction.

Vavr.io : quelques monades

Try<T>

  • Deux valeurs possibles : Failure(Throwable) et Success(T)
  • Se comporte comme un try, capture les exceptions.
  • Equivalent d'un Either maniant des Throwable

Vavr.io : quelques monades

Future<T>

  • Représente une valeur qui sera potentiellement disponible dans le futur
  • Equivalent d'un Try asynchrone.
  • Résultat mémoizé : un deuxième appel retourne le résultat du premier
